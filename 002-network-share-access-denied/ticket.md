# Ticket: Unable to access department shared folder

## Ticket Details

**Ticket ID:** IT-002

**Priority:** Medium

**Category:** File Share / Access Issue

**Affected device:** WIN11-CLIENT01

**Affected resource:** `\\DC01\Departments`

**Reported by:** Helpdesk / User Support

**Status:** Open

## Original Request

Hello,

A user is unable to access the department shared folder from their workstation.

They can sign in to the computer normally and internet access appears to be working, but when they try to open the shared folder, Windows shows an access denied message.

The user said they need the folder to access common department documents. Other users on the team appear to be able to open the same shared folder without issues.

Could someone please check why this user cannot access the department share?

## Reported Symptoms

* Device name: `WIN11-CLIENT01`
* User can sign in with their domain account
* Network access appears to be working
* User can reach the server name
* Shared folder opens for other users
* Affected user receives an access denied message

## Business Impact

The user cannot access shared department files needed for daily work.

## Additional Notes

The issue appears to affect one user rather than the whole shared folder. Please verify whether the user should have access and confirm why access is being denied.
